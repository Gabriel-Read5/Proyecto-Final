Introducción

En la actualidad, el desarrollo de software se ha convertido en una herramienta fundamental para la gestión de información en distintos ámbitos. La implementación de sistemas informáticos permite optimizar procesos, mejorar la organización de datos y facilitar la toma de decisiones. En este contexto, el presente proyecto tiene como finalidad el desarrollo de un sistema de gestión de usuarios, el cual permite administrar de manera eficiente la información relacionada con el acceso y control de usuarios dentro de una aplicación.
Este trabajo se realiza como parte de la asignatura Programación III, integrando conocimientos adquiridos en programación orientada a objetos, manejo de bases de datos y uso de herramientas de desarrollo. Además, se toma en cuenta la planificación del proyecto siguiendo una estructura organizada que incluye objetivos, alcance, tecnologías y definición de entregables. De esta manera, se busca no solo desarrollar un sistema funcional, sino también aplicar buenas prácticas en el proceso de desarrollo de software.

---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

1. Portada del trabajo

El presente proyecto corresponde al desarrollo de un sistema modelo titulado Sistema de Gestión de Usuarios (SGU), elaborado por Gabriel R. como parte de su formación académica. En la portada del trabajo se incluyen los datos esenciales como el nombre del estudiante, matrícula, título del proyecto y la fecha de entrega, lo cual permite identificar correctamente el documento y su propósito dentro del contexto educativo.

2. Nombre del proyecto de software

El proyecto recibe el nombre de Sistema de Gestión de Usuarios (SGU), ya que describe de manera clara y representativa la funcionalidad principal del sistema. Este nombre refleja el objetivo de administrar y controlar la información de los usuarios dentro de una aplicación, facilitando su organización y manejo.

3. Tecnología para aplicar

Para el desarrollo del sistema se utilizan diversas tecnologías y herramientas que permiten su correcto funcionamiento. Entre ellas se encuentra el lenguaje de programación Java como base principal, acompañado de MySQL para la gestión de la base de datos. Además, se emplean entornos de desarrollo como NetBeans o IntelliJ IDEA, así como Git y GitHub para el control de versiones. El sistema se construye bajo el paradigma de programación orientada a objetos e implementa operaciones tipo CRUD para la manipulación de datos.

4. Objetivo del proyecto

El objetivo principal de este proyecto es desarrollar una aplicación que permita gestionar usuarios de manera eficiente, proporcionando funcionalidades como registro, inicio de sesión y administración de datos. Con esto se busca facilitar el control de acceso y la organización de la información dentro de un sistema, ofreciendo una solución funcional y práctica.

5. Alcance del proyecto

El sistema incluye funcionalidades esenciales como el registro de usuarios, validación de credenciales, visualización de información, así como la edición y eliminación de datos. Todo esto se presenta a través de una interfaz gráfica sencilla. Sin embargo, el proyecto no contempla características avanzadas como seguridad robusta, integración con servicios externos o implementación en la web, ya que está enfocado en un entorno local con fines académicos.

6. Cronograma del proyecto

El desarrollo del sistema se organiza mediante un cronograma que abarca diferentes etapas, incluyendo el análisis, diseño, implementación, pruebas y documentación. Cada fase se realiza en un tiempo determinado, permitiendo una planificación estructurada del proyecto y asegurando el cumplimiento de los objetivos establecidos dentro del plazo previsto.

7. Definición del primer Release

La primera versión del sistema incluye las funcionalidades básicas necesarias para su funcionamiento, tales como el registro de usuarios, el inicio de sesión, la visualización de datos y la gestión de los mismos mediante operaciones CRUD. En este contexto, se definen requerimientos funcionales como la capacidad de almacenar, validar, editar y eliminar información, así como requerimientos no funcionales relacionados con la facilidad de uso, el rendimiento del sistema y su compatibilidad con entornos locales.

Conclusión

El desarrollo del Sistema de Gestión de Usuarios permitió aplicar de manera práctica los conocimientos adquiridos durante el curso, especialmente en el uso de tecnologías como Java, bases de datos y control de versiones. A través de este proyecto, se logró diseñar e implementar una solución funcional capaz de gestionar usuarios mediante operaciones básicas como registro, inicio de sesión, edición y eliminación de datos.

En conclusión, este trabajo representa una experiencia significativa en el desarrollo de software, fortaleciendo habilidades técnicas y de organización en el proceso.
